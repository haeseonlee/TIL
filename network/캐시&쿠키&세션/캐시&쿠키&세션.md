# 캐시&쿠키&세션

## 캐시&쿠키&세션이 필요한 이유
- HTTP 프로토콜은 기본적으로 상태를 유지하지 않는(stateless) 프로토콜이다. 즉, 이전의 요청과 상태 정보를 독립적으로 처리하지 않고, 이러한 상태 유지의 한계를 극복하기 위해 쿠키, 세션 등의 개념이 사용된다.

## 캐시(Cache)
- 웹 페이지를 구성하는 정적 파일(CSS, 이미지, 자바스크립트 등)을 사용자의 컴퓨터에 저장해 두고 재사용하는 기술이다. 서버의 네트워크 대역폭을 아끼고 브라우저 로딩 속도를 획기적으로 높여준다.

### 동작 방식
1. 브라우저가 서버에 특정 웹 페이지(예: 메인 로고 이미지)를 요청한다. 
2. 서버가 응답할 때 HTTP 헤더에게 캐시 유효 기간(Cache-Control)을 설정하여 이미지를 보낸다. 
3. 브라우저는 이 이미지를 로컬 캐시 저장소에 보관한다.
4. 이후 사용자가 같은 페이지를 다시 방문하면, 브라우저는 서버에 요청하지 않고 로컬 캐시에서 이미지를 즉시 불러와 화면에 띄운다.

### 장점
- 반복적인 요청을 줄여 더 빠른 응답 시간을 제공하고, 서버의 부하를 줄인다. 

### 단점
- 캐시된 데이터가 업데이트되지 않거나 유효 기간이 만료되지 않으면 최신 데이터를 얻을 수 없다. 
- 캐시 관리에 신경을 써야한다. 

### 주의할 점
- 캐시 헤더를 적절히 설정하여 클라이언트와 서버 간의 데이터 일치성을 유지해야 한다.
- 중요한 데이터나 동적인 컨텐츠에는 캐싱을 적용하지 않아야 한다. 

## 쿠키(Cookie)
- 클라이언트의 웹 브라우저에 저장되는 작은 텍스트 파일로, 웹 사이트에서 사용자의 정보를 저장하고 추적하는데 사용된다.
- HTTP의 비연결성(stateless)를 보완하기 위해 사용된다. 

### 동작 방식
1. 사용자가 로그인 버튼을 누르거나 사이트에 접속한다. 
2. 서버는 사용자가 필요한 정보를 담은 쿠키를 생성하여 HTTP 응답 헤더(Set-Cookie)에 포함한다. 
3. 브라우저는 이 쿠키를 로컬에 저장한다. 
4. 이후 브라우저가 해당 서버에 다시 요청을 보낼 때마다, 저장된 쿠키를 HTTP 요청 헤더에 자동으로 포함해서 전송한다. 
5. 서버는 쿠키를 읽어 사용자의 상태나 설정의 파악하고 맞춤형 페이지를 제공한다. 

### 장점
- 쉽게 구현 가능하며, 서버에 상태 정보를 저장하여 로그인 상태 등을 유지할 수 있다. 

### 단점
- 클라이언트에 저장되므로 보안에 취약할 수 있다. 
- 용량 제한이 있어 많은 데이터를 저장하기에는 제한적이다. 

### 주의할 점
- 민감한 정보(비밀번호 등)은 저장하지 않아야 한다. 
- 쿠키의 크기를 적절하게 관리하여 불필요한 데이터 전송을 방지해야 한다. 

## 세션(Session)
- 서버 측에서 사용자 상태를 추적하는 기술이다. 서버는 고유한 세션 ID를 클라이언트에게 부여하고, 해당 ID를 사용하여 상태를 유지한다. 
- 쿠키의 낮은 보안성을 보완하며, 로그인 정보처럼 유출되면 안되는 민감한 데이터를 브라우저가 아닌 웹 서버 측에서 안전하게 관리한다. 

### 동작 방식
1. 사용자가 아이디와 비밀번호를 입력하여 로그인을 요청한다.
2. 서버는 회원 정보가 올바른지 확인 후, 서버의 메모리(또는 DB)에 세션 ID를 생성하고 사용자의 로그인 정보를 저장한다. 
3. 서버는 발급한 세션 ID만 쿠키 또는 URL 매개변수에 담아 브라우저로 응답한다. 
4. 브라우저는 이 세션 ID가 담긴 쿠키 또는 URL 매개변수를 저장해둔다.
5. 이후 사용자가 다른 페이지를 요청할 때, 세션 ID 쿠키를 서버로 함꼐 보낸다. 
6. 서버는 전달받은 세션 ID로 서버의 세션 저장소를 조회하여 사용자를 식별하고 인증을 유지한다. 

### 장점
- 클라이언트에 저장되는 쿠키와 달리 서버에 저장되므로 보안에 더 강하다. 
- 크기 제한이 없어 많은 데이터를 저장할 수 있다.

### 단점
- 세션 데이터를 서버에 저장하므로 서버 자원을 소비한다.
- 세션 ID를 클라이언트에게 전달해야 하므로 약간의 오버헤드가 발생할 수 있다. 

### 주의할 점
- 세션 데이터를 안전하게 저장하고 관리해야한다. 
- 서버의 자원을 효율적으로 사용하기 위해 세션 만료 시간을 적절히 설정해야한다.
