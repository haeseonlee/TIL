# TCP(전송 제어 프로토콜)

## TCP란?
- 인터넷상에서 데이터를 안전하고 정확하게 전달하기 위해 사용하는 연결 지향형 프로토콜이다.
- 송신측과 수신측이 서로 연결된 상태를 확인하고, 데이터 중간에 유실되거나 순서가 바뀌지 않도록 보장해 주는 역할을 한다.

## TCP의 핵심 특징
- **연결 지향형 (Connection-oriented)**
  - 데이터를 주고받기 전에 송신측과 수신측이 서로 연결을 설정한다. 이떄 3-Way Handshake라는 과정을 거친다. 
- **흐름 제어 (Flow Control)**
  - 수신측의 데이터 처리 속도에 맞춰 송신측의 데이터 전송량을 조절한다.(수신측의 버퍼가 넘치는 것을 방지)
- **혼잡 제어 (Congestion Control)** 
  - 네트워크가 혼잡한 상태(네트워크 내의 데이터 양이 과도할 때)라고 판단되면 송신측의 전송 속도를 줄여 네트워크 붕괴를 막는다. 
- **높은 신뢰성 보장**
  - 데이터가 중간에 유실되면 재전송을 요청하고, 순서가 뒤바뀌어 도착하면 올바른 순서로 재조립한다. 

## TCP 통신 과정
- TCP 통신을 하려면 네트워크 설정이 먼저 필요하다. 데이터를 발송하는 애플리케이션, 데이터를 수신하는 애플리케이션 모두 준비가 되었다는 보장하기 위해서 필요하다. 데이터를 정확하게 전달하기 위해 세션을 만드는 중요한 과정이다. 
- TCP 통신은 크게 **연결 수립, 데이터 송수신 및 흐름/혼잡 제어, 연결 해제** 3단계로 나뉜다.

### 연결 수립 단계(3-Way HandShake & 소켓 상태 변화)
  - **SYN(클라이언트->서버)**: 클라이언트가 서버에게 SYN 플래그가 설정되니 메세지를 보낸다. 이떄 SYN은 클라이언트의 임의 시퀀스 번호 A를 포함한다.
  - **SYN-ACK(서버->클라이언트)**: 서버는 클라이언트의 SYN에 응답하여 SYN-ACK 메세지를 보낸다. 서버는 자신만의 새로운 시퀀스인 B를 포함한 SYN을 전송하고, 클라이언트의 시퀀스 번호 A에 1을 더한 ACK을 함께 보낸다.(A+1)
  - **ACK(클라이언트->서버)**: 클라이언트는 서버로부터 받은 SYN에 대해 ACK 메세지를 전송한다. 이떄 ACK는 서버의 시퀀스 번호 B에 1을 더한 값(B+1)을 포함한다. 

### 데이터 송수신 및 흐름/혼잡 제어
- 연결이 이루어지면 애플리케이션 계층의 데이터가 TCP 세그먼트 단위로 쪼개져서 전송된다. 
  - 데이터 분할 및 버퍼링: 전송할 데이터를 세그먼트 단위로 쪼개어 송신 버퍼에 저장한 뒤 순차적으로 보낸다. 
  - 순서 보장 및 검증: 각 패킷에 순서 번호를 부여하고, 수신자가 데이터를 받을 때마다 응답(ACK)을 보낸다.
  - 신뢰성 제어: ACK 응답이 누락되면 데이터가 유실된 것으로 간주하여 패킷을 재전송한다.
  - 흐름 및 혼잡 제어: 수신 측의 버퍼 여유 공간과 네트워크 상태를 파악하여 송신 속도를 조절한다.

### 연결 해제 단계(4-Way Handshake & TIME_WAIT)
- TCP는 전이중 통신을 지원하므로, 한쪽이 연결을 끊자고 해도 반대쪽에서는 아직 보낼 데이터가 남아있을 수 있다. 따라서 연결을 끊을 때는 4단계를 거쳐 양방향을 각각 독립적으로 닫는다.
  - FIN(클라이언트->서버): 클라이언트가 송신을 마쳤음을 알리기 위해 FIN 패킷을 보낸다. 클라이언트는 FIN_WAIT_1 상태가 된다.
  - ACK(서버->클라이언트): 서버는 FIN을 받고 확인 응답(ACK)을 보낸다. 서버는 아직 남은 데이터를 보내기 위해 CLOSE_WAIT 상태가 되며, 클라이언트는 이 ACK을 받고 FIN을 기다리는 FIN_WAIT_2 상태가 된다.
  - FIN(서버->클라이언트): 서버도 데이터를 모두 보냈으면, 클라이언트에게 연겨을 종료하겠다는 FIN 패킷을 보내고 LAST_ACK 상태로 돌아간다.
  - ACK(클라이언트->서버): 클라이언트는 서버의 FIN을 확인하고 마지막 ACK를 보낸 뒤, 소켓을 즉시 닫지 않고 TIME_WAIT 상태로 전환된다. 서버는 이 ACK를 닫으면 소켓을 완전히 닫는 CLOSED 상태가 된다.
