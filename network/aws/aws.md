# AWS

## AWS(Amazon Web Service)란?
- AWS는 아마존에서 제공하는 클라우드 컴퓨팅 플랫폼이다. 전 세계적으로 가장 널리 사용되는 클라우드 서비스이다.
- 기업이나 개인이 직접 물리적인 서버 장비를 사서 구축할 필요 없이, 인터넷을 통해 컴퓨팅 파워, 스토리지(저장 공간), 데이터베이스 등 다양한 IT 인프라를 빌려쓸 수 있게 해준다.

## 알아두면 좋은 개념들
### VPC

- 클라우드 인프라 내에 격리된 독립된 네트워크 공간이다. 이 공간 안에서 사용할 IP 주소 범위를 먼저 지정하게 된다.
- Virtual Private Cloud(가상 사설 클라우드)는 쉽게 말해 클라우드(AWS)라는 거대한 공용 공간 안에 나만을 위해 분리한 안전한 개인 방을 만드는 기술로 이를 활용하면 방 안에 벽을 세우고, 문을 달고, 허락된 사람만 들어오게 제어할 수 있다.

### Subnet

- 하나의 거대한 네트워크를 논리적으로 분할한 더 작은 네트워크 영역을 의미한다. 원래 인테넷 구조에서 수억 대의 컴퓨터가 하나의 커다란 네트워크에 통쨰로 묶여있다면, 특정 컴퓨터를 찾기 위해 모든 컴퓨터에 신호를 보내야 하는 비효율이 발생한다. 이를 해결하기 위해 관리하는 행위를 서브네팅이라고 하며, 그렇게 쪼개진 개별 네트워크가 서브넷이다.
    - **퍼블릭 서브넷(Public Subnet):** 사용자들이 인터넷을 통해 직접 접속해야하는 웹 서버나 프론트엔드 파일들이 여기에 위치합니다.
    - **프라이빗 서브넷(Private Subnet):** 외부에 절대 노출되면 안되는 핵심 비즈니스 로직 서버(WAS)나 고객 정보, 결제 정보 등이 담긴 데이터베이스가 여기에 위치합니다.
- 서브넷을 쪼개서 얻는 이점?
    - 만약 홈페이지 화면을 보여주는 웹 서버(퍼블릭 서브넷)이 해킹을 당하더라도, 핵심 데이터가 담긴 DB 서버는 프라이빗 서브넷이라는 별도의 구역에 격리되어 있기 때문에 해커가 쉽게 침투할 수 없다. 즉, 한 구역이 뚫려도 전체 시스템이 무너지는 것을 막아준다.
    - 각 서브넷마다 라우팅 테이블(이정표)를 다르게 쥐여줄 수 있다. 퍼블릭 서브넷의 이정표에는 외부 인터넷으로 가려면 정문(인터넷 게이트웨이)으로 가라고 적어둔다. 프라이빗 서브넷의 이정표에는 인터넷 게이트웨이로 가는 길을 아예 적어두지 않는다.

### Amazon Route 53

- 우리가 웹 브라우저 주소창에 naver.com 이나 google.com 같은 문자 주소를 입력했을 때, 컴퓨터가 이해할 수 있는 실제 숫자로 된 IP 주소로 번역해서 올바른 목적지로 안내해주는 클라우드 기반의 DNS(Domain Name System) 서비스이다.
- 핵심 기능
    - **DNS 라우팅** : 친구에게 전화를 걸 때, 우리는 친구의 이름(도메인)을 검색하지 010-xxxx-xxxx라는 번호(IP 주소)를 매번 외워서 누르지 않는다. 휴대폰 연락처가 이름과 번호를 매칭해 주듯, Route 53은 mysite.com 을 치면 그 사이트의 실제 주소는 `54.23.11.22`입니다"라고 연결해준다.
    - **트래픽 라우팅** : Route 53은 단순히 주소를 찾아주는 것을 넘어, 사용자를 가장 빠르고 안전한 길로 안내하는 내비게이션 역할도 한다.
    - **상태 검사** : Route 53은 내가 만든 서버들이 현재 살이있는지 죽어있는지 실시간으로 감시한다. 만약 메인 서버가 고장나면 사용자들이 에러화면을 보지 않도록 정상 작동중인 예비 서버로 주소를 즉시 변경하여 연결해준다.

### Internet Gateway

- VPC 내부의 서버들이 인터넷 세상과 데이터를 주고받을 수 있도록 통로를 열어주는 핵심 컴포넌트이다.
- 인터넷 게이트웨이는 양방향 통신을 모두 지원한다. 퍼블릭 서브넷에 있는 웹 서버가 외부 사용자에게 홈페이지 화면을 보여줄 때도, 사용자가 웹 사이트에 접속하기 위해 들어올 때도 모두 이 문을 통과해야한다.
- 라우팅 테이블과의 관계 : 서브넷이 퍼블릭이냐 프라이빗이냐를 결정짓는 유일한 기준은 그 서브넷의 이정표(라우팅 테이블)가 인터넷 게이트웨이를 가리키고 있는가? 이다.

### Route Table

- 라우팅 테이블은 목적지와 타겟이라는 두 가지 정보가 적힌 규칙들의 리스트이다.
- 쉽게 말해 네트워크 세상의 이정표이자 내비게이션 안내 지도이다. VPC라는 큰 아파트 단지를 만들고 그 안에서 서브넷이라는 여러 구역을 나누어 놓아도, 데이터(트래픽)가 어디로 가야 하는지 알려주는 안내판이 없다면 데이터들은 길을 잃고 헤매게 된다. 라우팅 테이블은 이 주소로 가고싶다면, 저쪽 문으로 가세요라고 규칙을 적어놓은 지도이다.

### EC2

- 쉽게 말해 인터넷 세상에서 컴퓨터를 필요한 만큼 빌려 쓰는 서비스이다. 우리가 개발한 프로그램을 24시간 안정적으로 돌리려면 항상 켜져 있는 컴퓨터가 필요하다. 예전에는 거대한 서버 장비를 직접 돈 주고 사서 전산실에 설치해야 했지만, EC2를 사용하면 클릭 몇 번만으로 몇 초 만에 사양이 좋은 컴퓨터를 사지 않고 빌려서 쓸 수 있다.
