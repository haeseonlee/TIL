# OSI 7 계층

## OSI 7 계층이란?
서로 다른 시스템이 통신할 수 있도록 국제표준화기구(ISO)에서 만든 표준 네트워크 모델이다.
복잡한 통신 과정을 7개로 나누어 놓았기 때문에, 문제가 발생하였을 때 다른 단게의 장비 및 소프트웨어를 건들지 않고도 이상이 생긴 단계만 고칠 수 있다.

### 1. 물리 계층(Physical Layer)
- **역할**: 데이터를 물리적인 신호(전기 신호, 광신호 등)으로 변환하여 케이블을 통해 실제로 전송한다.
- **설명**: 오직 0과 1로 이루어진 비트(Bit) 데이터를 전선이나 무선 주파수로 실어 나르는 층이다. 데이터가 무엇을 의미하는지 전혀 알지 못하며, 단순히 신호를 주고받는 일을 한다.
- **전송 단위**: 비트(Bit)
- **장비**: 케이블, 리피터, 허브

### 2. 데이터 링크 계층(Data Link Layer)
- **역할**: 물리적인 맥(MAC) 주소를 사용하여 인접한 노드 간의 신뢰성 있는 데이터 전송을 보장한다. 
- **설명**: 네트워크 계층이 가야할 주소를 알려주면, 데이터 링크 계층은 바로 옆에 있는 장비(노드)로 데이터를 안전하게 전달한다. 물리계층에서 발생할 수 있는 오류를 잡고 흐름을 제어한다.
- **전송 단위**: 프레임(Frame)
- **프로토콜**: 이더넷(Ethernet), Wi-Fi, PPP
- **장비**: 스위치, 브릿지, 랜카드(NIC)

### 3. 네트워크 계층(Network Layer)
- **역할**: 데이터를 목적지까지 가장 안전하고 빠르게 전달하는 경로를 찾는다. (라우팅)
- **설명**: 이 계층에서 가장 중요한 것은 IP 주소이다. 전 세계 수 많은 네트워크 중 목적지 컴퓨터가 어디에 있는지 주소를 확인하고, 최적의 길을 안내하는 내비게이션 역할을 한다.
- **전송 단위**: 패킷(Packet)
- **프로토콜**: IP, ICMP, ARP
- **장비**: 라우터, L3 스위치

### 4. 전송 계층(Transport Layer)
- **역할**: 송신자와 수신자 간의 신뢰성 있고 정확한 데이터 전송을 보장한다.
- **설명**: 데이터를 보내기 적당한 크기(Segment)로 쪼개고, 받는 쪽에서 순서대로 잘 조립할 수 있게 번호를 매긴다. 데이터가 유실되면 재전송을 요청(흐름 제어, 오류 제어)하는 중요한 계층이다.
- **전송 단위**: 세그먼트(Segment)
- **프로토콜**: TCP, UDP
- **장비**: 게이트웨이, L4 스위치

### 5. 세션 계층(Session Layer)
- **역할**: 양 끝단의 응용 프로세스가 통신을 관리하기 위한 설정을 유지하고 동기화한다.
- **설명**: 통신을 하기 위한 대화 통로(세션)을 만들고 유지하다가 통신이 끝나면 끊어내는 관리자이다. 통신 중 에러가 나면 처음부터 다시 보내지 않고 끊긴 지점부터 다시 보낼 수 있도록 동기점을 설정하는 역할을 한다.
- **프로토콜/기능**: SSH, TLS, 세션 복구 및 동기화

### 6. 표현 계층(Presentation Layer)
- **역할**: 응용 계층으로부터 받은 데이터를 하위 계층이 이해할 수 있는 공통 형식으로 변환하거나, 그 반대의 작업을 수행한다.
- **설명**: 데이터 번역 역할을 한다. 인코딩/디코딩, 암호화/복호화, 데이터 압축 등이 여기서 일어난다. (예: 텍스트를 UTF-8로 변환하거나, 이미지를 JPEG로 압축)
- **확장자/기술**: JPEG, MPEG, ASCII, SSL/TLS

### 7. 응용 계층(Application Layer)
- **역할**: 사용자가 네트워크에 접근할 수 있도록 인터페이스를 제공한다. 우리가 사용하는 프로그램들이 이 계층에서 동작한다.
- **설명**: 웹 브라우저나 이메일 등이 대표적이며, 사용자에게 직접 보이면서 데이터를 생성하는 출발점이다. 
- **프로토콜**: HTTP(웹), FTP(파일 전송), SMTP(이메일), DNS(도메인 주소 변환)

## 데이터 흐름(Data Flow)
컴퓨터가 데이터를 보낼 때는 7계층에서 1게층으로 내려가면서 각 게층의 정보가 붙는 **캡슐화(Encapsulation)** 과정을 거치고, 받을 때는 1계층에서 7계층으로 올라가면서 헤더를 하나씩 벗겨내는 **역캡슐화(Decapsulation)** 과정을 거친다.

### 캡슐화(Encapsulation)
사용자가 컴퓨터에서 "안녕하세요"라는 메세지를 보내면, 데이터는 상위 계층에서 하위 계층으로 내려가며 각 계층의 기능 수행에 필요한 정보가 담긴 헤더(Header)를 붙인다. 이 과정을 캡슐화라고 한다.

- **7계층 ~ 5계층(응용,표현,세션)**: 사용자가 작성한 순수한 데이터가 생성된다.
- **4계층(전송 계층)**: 데이터가 너무 크면 쪼갠 후 앞에 L4 헤더를 붙인다. 여기에는 데이터를 정확한 프로그램으로 찾아가게 해주는 출발지/목적지 포트(Port)번호와 데이터 순서 등이 담긴다.
  - **결과물**: 세그먼트
- **3계층(네트워크 계층)**: 세그먼트 앞에 L3 헤더를 붙인다. 여기에서는 전세계 수많은 컴퓨터중 목적지를 정확히 찾아갈 수 있도록 출발지/목적지 IP 주소가 담긴다.
  - **결과물**: 패킷
- **2계층(데이터 링크 계층)**: 패킷 앞뒤로 L2 헤더와 트레일러를 붙인다. 헤더에는 바로 다음 물리적 장치로 가기 위한 출발지/목적지 MAC 주소가 담기고, 트레일러에는 데이터 전송 중 에러가 났는지 검사하기 위한 값(FCS)이 담긴다.
  - **결과물**: 프레임
- **1계층(물리 계층)**: 완성된 프레임(0과 1로 구성된 이진 데이터)을 랜선이나 와이파이 같은 전송 매체로 보낼 수 있도록 전기 신호나 광신호(Bit)로 변환하여 상대방에게 보낸다.

### 역캡슐화(Decapsulation)
수신 측 컴퓨터는 신호를 받으면, 송신 측의 반대 순서로 헤더를 하나씩 읽고 벗겨내며 상위 계층으로 올린다. 각 계층은 자신과 매칭되는 헤더 정보만 확인하고 지워버린다.

- **1계층(물리 계층)**: 케이블을 통해 들어온 전기 신호를 다시 0과 1의 비트(Bit) 데이터로 복원하여 2계층으로 올린다.
- **2계층(데이터 링크 계층)**: L2 헤더를 확인하여 목적지 MAC 주소가 내 랜카드 주소와 맞는지 확인한다. 일치하면 트레일러로 에러 검사를 한 뒤, L2 헤더와 트레일러를 떼어내고 패킷을 3계층으로 올린다.
- **3계층(네트워크 계층)**: L3 헤더를 확인하여 목적지 IP 주소가 내 IP가 맞는지 확인한다. 내 컴퓨터로 온게 맞다면 L3를 떼어내고 세그먼트를 4계층으로 올린다.
- **4계층(전송 계층)**: L4 헤더를 확인하여 쪼개진 데이터들을 순서대로 조립하고, 이 데이터가 어떤 프로그램으로 가야하는지 포트 번호를 확인한다. 목적지가 확인되면 L4 헤더를 떼어내고 순수 데이터를 상위 계층으로 올린다.
- **5계층 ~ 7계층(세션,표현,응용)**: 데이터의 암호화를 풀거나 압축을 해제하여, 사용자가 최종적으로 화면에서 "안녕하세요"를 볼 수 있도록 처리한다.  

**역할의 분리:** 각 계층은 자신이 맡은 헤더 정보만 신경쓰면 된다. 예를 들어, 3층 라우터는 IP 주소(L3 헤더)만 확인하고 길을 찾으면 되지 이 데이터가 카카오톡 메세지인지 유트브 영상인지는 알 필요가 없다.  

**유지보수의 편리함:** 만약 와이파이에서 유선 랜선으로 인터넷 연결 방식이 바뀌더라도 (1, 2계층의 변화), 상위 계층인 웹 브라우저(7계층)나 TCP(4계층) 프로그램은 아무런 수정 없이 그대로 작동할 수 있다.
